# Founder-Led Discovery Spine

This skill has two layers:

- `SKILL.md`: the general founder-led discovery framework
- `profiles/wpatent.md`: the W&Patent-specific operating profile

Supporting files:

- `profiles/wpatent-proof-network.md`: W&Patent proof-network inventory and gap map
- `profiles/wpatent-proof-prompts.json`: W&Patent proof-prompt tiers for broad discovery, narrow proof validation, and branded identity-retrieval control checks
- `profiles/openfor-comparison-control.json`: comparison control profile for testing the framework against another founder-led site
- `runbooks/founder-discovery-publishing-loop.md`: agent-facing loop for turning internal measurement into founder-facing public assets
- `templates/page-brief-template.md`: reusable citation-surface page brief
- `briefs/startup-patent-strategy.md`: first concrete W&Patent page brief
- `briefs/patent-commercialization-for-founders.md`: second concrete W&Patent page brief
- `runbooks/wpatent-on-demand-operations.md`: lightweight agent-ready operating runbook
- `test-prompts.json`: W&Patent prompt clusters and validation prompts

## How To Use It

1. Start with `SKILL.md` for the five-part model and the evidence hierarchy.
2. Load `profiles/wpatent.md` when working specifically on W&Patent.
3. Load `profiles/wpatent-proof-network.md` when the work involves proof gaps, supporting assets, or offsite reinforcement.
4. Load `profiles/wpatent-proof-prompts.json` when the work involves proof-legibility checks, narrow prompt pilots, or branded identity-retrieval controls.
5. Load `profiles/openfor-comparison-control.json` when you want a troubleshooting control outside W&Patent.
6. Load `runbooks/founder-discovery-publishing-loop.md` when the task is to turn internal measurement into founder posts, case notes, proof notes, or other public-facing assets.
7. Use `templates/page-brief-template.md` when drafting or reviewing new citation-target pages.
8. Start with `briefs/startup-patent-strategy.md` when drafting the first priority page.
9. Use `runbooks/wpatent-on-demand-operations.md` when you want an agent to run the discovery loop on demand.
10. Use `test-prompts.json` for prompt-cluster mapping and rerun checks.

## W&Patent Role

The W&Patent profile is not a separate framework.
It is an operating profile layered onto the broader Founder-Led Discovery Spine.

It adds:

- current discovery baseline
- current proof-network baseline
- public publishing loop guidance
- named topic clusters
- page-priority recommendations
- concrete page-brief expectations
- rerun and validation rules
